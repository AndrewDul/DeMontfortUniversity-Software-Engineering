# Web Application Development - HTML Basics

This project covers the fundamentals of HTML, focusing on the default flow of elements, accessibility, and semantic structure. It includes practical exercises to understand how HTML elements behave by default, how to structure a document semantically, and how to ensure accessibility for all users.

## Tasks and Requirements

### 1. **Default Flow of Elements**
   - Understand how block and inline elements behave by default.
   - Experiment with simple text, block elements (`<div>`, `<p>`), and inline elements (`<span>`, `<a>`).
   - Observe how images (`<img>`) behave as inline-block elements.
   - Use browser developer tools to inspect default styles (e.g., margins, padding, font properties).

### 2. **Semantic Structure**
   - Use semantic HTML elements like `<header>`, `<main>`, `<footer>`, and `<section>` to structure the document.
   - Add headings (`<h1>` to `<h6>`) to create a logical hierarchy.
   - Use `<figure>` and `<figcaption>` for images with captions.
   - Include a form with accessible inputs using `<label>`, `<input>`, and `<fieldset>`.

### 3. **Accessibility**
   - Ensure all images have `alt` attributes for screen readers.
   - Use ARIA roles and attributes where necessary (e.g., `aria-labelledby`, `aria-level`).
   - Test the document with a screen reader to ensure proper navigation.
   - Validate the HTML using an online validator.

### 4. **Final Implementation**
   - Combine all tasks into a single HTML document.
   - Add basic CSS for styling (e.g., margins, padding, fonts).
   - Include JavaScript for interactive elements (e.g., form validation).

---

## Files

1. **`index.html`**: The main HTML file containing the structure and content.
2. **`styles.css`**: The CSS file for styling the document.
3. **`script.js`**: The JavaScript file for adding interactivity.

---

## How to Run

1. Clone the repository or download the files.
2. Open `index.html` in a web browser.
3. Use the browser's developer tools to inspect elements and test accessibility.
4. Validate the HTML using an online validator (e.g., [W3C Validator](https://validator.w3.org/)).
